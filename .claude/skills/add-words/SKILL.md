# Add Words Skill

사용자가 새로운 일본어 단어를 추가하고자 할 때 사용하는 skill입니다.

## 입력 형식

사용자는 다음과 같은 형태로 데이터를 제공합니다:
- 파일명: `{num}.json` (예: 15.json)
- 데이터: `["단어1", "단어2", ...]` JSON 배열

## 처리 규칙

### 1. 문자 타입 판별
- **히라가나만**: 완전히 무시 (저장하지 않음)
- **카타카나 포함**: `katakana` 배열에 추가
- **한자 포함**: `kanji` 배열에 추가

### 2. 출력 포맷

```json
{
  "kanji": [
    { "prob": "히라가나 읽기", "ans": "한자 단어" }
  ],
  "katakana": [
    { "ans": "카타카나", "prob": "한국어 뜻" }
  ]
}
```

### 3. 각 타입별 처리

#### 한자가 포함된 단어 (kanji)
- `prob`: 단어의 히라가나 읽기 (Claude가 직접 작성)
- `ans`: 원본 단어 (한자 포함)

#### 카타카나 단어 (katakana)
- `ans`: 원본 카타카나 단어
- `prob`: 한국어 뜻 (Claude가 직접 작성)

## 실행 절차

1. 사용자가 제공한 단어 목록을 분석
2. 히라가나만 있는 단어는 제외
3. 카타카나/한자 단어를 분류
4. 한자 단어의 경우: Claude가 히라가나 읽기를 직접 작성
5. 카타카나 단어의 경우: Claude가 한국어 뜻을 직접 작성
6. `/Users/joseonghyeon/nihonki/public/data/{num}.json`에 바로 저장
7. 원본 입력 파일 삭제
8. `/Users/joseonghyeon/nihonki/src/screens/Home/Home.tsx`의 `lessons` 배열에 새 단원 번호 추가 (없는 경우에만, 오름차순 유지)

## 문자 타입 판별 기준

- 히라가나: U+3040-U+309F (ぁ-ゟ)
- 카타카나: U+30A0-U+30FF (゠-ヿ)
- 한자: U+4E00-U+9FFF, U+3400-U+4DBF

## 예시

입력:
```
15.json
["勉強する", "チャレンジ", "たべる", "コンピュータ"]
```

처리:
- `勉強する` → kanji (한자 포함)
- `チャレンジ` → katakana
- `たべる` → 무시 (히라가나만)
- `コンピュータ` → katakana

Claude가 작성:
- `勉強する` → 읽기: べんきょうする
- `チャレンジ` → 뜻: 도전
- `コンピュータ` → 뜻: 컴퓨터

출력 (15.json):
```json
{
  "kanji": [
    { "prob": "べんきょうする", "ans": "勉強する" }
  ],
  "katakana": [
    { "ans": "チャレンジ", "prob": "도전" },
    { "ans": "コンピュータ", "prob": "컴퓨터" }
  ]
}
```
